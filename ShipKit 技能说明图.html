编排层 · Orchestration Layer
ship-orchestrator 必需
Orchestrator
统一入口和路由中心。识别 feature 开发场景，路由到对应阶段技能，维护 meta.yml 单一事实源，并执行唯一的用户确认门禁（Design → Build）。
核心职责
统一入口和路由 4 种场景识别 状态推进和阻塞恢复 Design → Build 用户确认门禁 Design 模板意图传递
核心阶段 · Core Stages
ship-understand 必需
Core Stage
理解需求阶段。加载 spec 上下文，解析需求/PRD/原型，必要时调用 grill-me 质询，产出 requirements.md。
产出
spec 上下文加载 需求解析 requirements.md Understand 阶段 grill-me
ship-design 必需
Core Stage
技术设计阶段。基于 requirements.md 产出合一技术设计：API Contract、数据模型、前后端方案，并在 Build 前等待用户确认。
产出
API Contract 数据模型 前后端方案合一 Design 阶段 grill-me 用户确认前置 Design Reference Template 模板偏离记录
ship-build 必需
Core Stage
实现与测试阶段。用户确认设计后，生成任务清单，实现代码，运行测试，验证 AC，产出 verification.md。
产出
build-plan.yml 实现顺序 自动化测试 AC 验证报告 spec 完成后更新
支持层 · Support Layer
ship-split
Optional Pre-stage
可选前置阶段。拆分大需求，生成 splits.yml，追踪依赖，支持 TAPD/Jira 集成框架。
功能
大需求拆分 splits.yml TAPD/Jira 集成框架 依赖追踪
ship-spec 必需
Tool
工具技能。管理 .docs/spec 知识库，stage hooks，多项目隔离，规范沉淀评分，处理 spec 与模板边界。
功能
.docs/spec 知识库 stage hooks 多项目隔离 规范沉淀评分 spec 与模板边界
ship-grill-me 必需
Embedded Helper
嵌入式质询助手。Understand/Design ready 前发现 blocking 问题，先查仓库和 spec，再一次只问用户一个关键问题。
功能
精确触发矩阵 阻塞问题识别 先查仓库/spec 再问用户 一次只问一个问题 Design 模板 blocking 审查

单一事实源：meta.yml

Feature 状态以 .docs/feature-*/meta.yml 为单一事实源，所有阶段技能读写此文件来维护状态一致性。

feature_name 功能名称
current_stage understand | design | build | done
status in_progress | blocked | completed
scenario quick_start | full_flow | prd_direct | split_first
created_at / updated_at 时间戳
spec_refs 引用的规范列表
requested_design_template 用户显式模板意图原话
design_template_ref Design 阶段解析后的模板引用
artifacts 产出文件路径
