# PRD Review Scoring

## 目标

对 PRD 做证据驱动的评审评分，输出结构化中文 md 报告。核心不是复述需求，而是判断这份需求是否足够进入设计、开发和验收。

## 输入

- PRD/BRD/需求文档路径、文本、截图或链接。
- 可选：当前代码仓库、业务域、目标系统、评审口径或输出路径。
- 默认报告输出到 `.docs/`，文件名使用 `prd-review-<topic>-YYYY-MM-DD.md`。

## 必须遵守

- 文档和报告使用中文，保留 technical English names、路径、API 名。
- 先查证据再评分；不能凭印象给分。
- 不问仓库能回答的问题；先自行读取 PRD、搜索仓库、检查相关代码/文档。
- 如需外部对标，优先使用官方文档、产品帮助中心、法规原文、厂商公告等一手资料，并在报告中写明链接。
- 仓库没有相关业务能力时，要明确写成“未发现相关能力证据”，不能把不存在的能力推断为已有基础。
- 若 PRD 涉及代码改动建议，评审报告只评审与建议，不直接改业务代码。

## 工作流

1. **PRD 解析**
   - 抽取版本、背景、目标、角色、范围、业务流程、功能点、状态流转、验收指标、非功能要求。
   - 建立需求编号，至少覆盖端侧、后台、AI 生成链路、内容审核、配置、监控、合规与数据。
   - 记录输入证据：文件路径、抽取方式、抽取时间、关键章节。

2. **当前仓库勘查**
   - 用 `rg --files`、`rg`、配置文件和文档检查技术栈、现有模块、相似实现、测试/验证工具。
   - 只勘查与当前 PRD 相关的能力，例如直播、回放、ASR、PPT、AI summary、prompt、审核、任务状态、监控、权限、配置。
   - 报告中区分：
     - `已发现能力`：必须有文件路径或命令证据。
     - `未发现能力`：写明搜索关键词和结论。
     - `不适用/当前仓库非业务仓库`：说明边界。

3. **业界类似产品调研**
   - 至少调研 4 个相关方向；优先覆盖 AI 会议总结、智能纪要、教育视频/课堂回放、视频章节/搜索、内容审核或生成式 AI 合规。
   - 每条对标必须包含：产品/来源、官方链接、可借鉴设计、对当前 PRD 的启示、是否影响评分。
   - 只把官方资料作为事实依据；二手文章最多作为线索，不作为评分证据。

4. **多维度评分**
   - 使用 0-10 分制，保留 1 位小数。
   - 默认维度：
     - 业务目标与成功指标
     - 用户价值与适老化体验
     - 功能链路完整性
     - 业务域覆盖完整性
     - 后台运营与内容审核闭环
     - AI 生成质量与可控性
     - 数据来源、模型输入与可追溯
     - 状态流转、异常处理与兜底
     - 非功能要求、监控与可运维性
     - 权限、安全、隐私与合规
     - 与当前仓库已有能力匹配度
     - 开发可实现性与改动影响范围
     - 可测试性与验收标准
   - 总分默认取加权平均。权重可按业务风险调整，但必须在报告中披露。

5. **多角色评审**
   - 架构师：系统边界、数据链路、状态机、扩展性、可靠性、成本。
   - 用户：任务可完成性、可理解性、适老化、无障碍、信任感。
   - 产品经理：场景闭环、优先级、MVP 边界、指标可验证、灰度策略。
   - 开发：接口、数据模型、端/后台改动范围、依赖、边界条件、测试成本。
   - QA：验收用例、异常路径、状态流转、兼容性、回归范围。
   - 运营/教务：审核发布、人工兜底、SOP、质量抽检、内容下线。
   - 合规/安全：个人信息、课堂内容授权、AI 生成标识、日志与审计。

6. **结论与行动项**
   - 给出评审结论：`通过`、`有条件通过`、`需要重大修订`、`暂不通过`。
   - 列出 Critical/Major/Minor 问题，每个问题必须有影响、证据、建议。
   - 给出下一步行动项，标明建议 owner、优先级、进入设计/开发前的 gate。

## 报告模板

优先使用 `assets/report-template.md`。如任务需要轻量报告，可压缩章节，但不能省略：

- 评审结论和总分
- 证据来源
- 仓库勘查
- 业界调研
- 多维度评分
- 多角色评审
- Top 问题和行动项

## 自检

报告完成前必须检查：

- 是否每个评分都有依据，而不是纯主观判断。
- 是否明确区分 PRD 原文、仓库证据、外部官方资料、评审推断。
- 是否覆盖端侧、后台、AI 生成、审核发布、配置、状态、监控、合规和测试。
- 是否指出 MVP 边界和进入下一阶段的 gate。
- 是否保存为 `.docs/` 下的 md 文件。
