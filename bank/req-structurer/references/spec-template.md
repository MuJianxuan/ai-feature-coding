# Spec 输出模板

本模板定义 req-structurer 输出的 spec 格式，正文严格对齐 brainstorming spec，确保 writing-plans 可直接消费。

---

## Spec 文件命名

保存到 `docs/superpowers/specs/YYYY-MM-DD-<feature-name>-design.md`

---

## 模板正文

以下为 spec 正文的章节结构。每个章节提供写作指引，具体内容由 Skill 根据需求生成。

---

# [功能名称] — 设计文档

## 概述

一段话说清楚三件事：做什么、给谁用、技术方案是什么。

**写作要求：**
- 不超过 200 字
- 必须包含：目标用户、核心功能、技术实现方向
- 不写背景铺垫，直接进入正题

## 页面/模块结构

每个页面或模块作为一个三级标题，分别描述。

### [页面/模块名称]

**包含元素：** 列出该页面的所有 UI 组件和内容区域。

**用户交互行为：**
- 用户可以做什么操作
- 每个操作的触发方式和效果

**状态变化：**
- 列出该页面可能出现的不同状态（如空状态、加载态、正常态、错误态）
- 每种状态下的展示内容

**写作要求：**
- 每个页面/模块独立描述，不依赖其他页面的上下文
- 交互行为用动词开头（"点击XX按钮→触发YY"）
- 状态变化要覆盖正常态和所有异常态

## 数据结构

涉及的模型定义。每个核心数据实体作为一个三级标题。

### [实体名称]

```json
{
  "fieldName": "示例值",
  "anotherField": "示例值"
}
```

**字段说明：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fieldName | string | 是 | 字段用途说明 |

**写作要求：**
- 用 JSON 示例 + 表格说明的双重格式
- 每个字段必须标注类型和是否必填
- 枚举类型列出所有可选值
- 关联关系用字段名 + 引用目标表示（如 `"userId": "关联 users.id"`）

## 业务规则

该功能涉及的所有业务逻辑。

**排序规则：** 数据的默认排序和用户可自定义排序。

**状态流转：** 如果实体有状态，用文字或表格描述状态转换路径和条件。

**计算规则：** 如果涉及计算，给出公式和精度要求。

**限制条件：** 频率限制、数量限制、时间限制等约束。

**写作要求：**
- 每条规则必须明确、无歧义，能直接转化为代码逻辑
- 不写"合理处理"、"适当限制"等模糊描述
- 计算公式用明确的数学表达式，不使用自然语言描述计算过程

## 异常与边界处理

覆盖所有已识别的异常场景和边界条件。

**错误处理：** 各类错误（网络、业务、数据）的用户感知和处理方式。

**空状态：** 无数据时的展示内容和引导操作。

**边界条件：** 数据量极值、输入边界、并发冲突等的处理方式。

**降级方案：** 外部依赖不可用时的应对策略。

**写作要求：**
- 每种异常场景必须给出具体的处理方式，不只是"友好提示"
- 边界条件用具体数字（如"超过1000条时分页加载"），不用"大量数据"等模糊表述

## 非功能性需求

### 性能要求
- 预期数据量级
- 响应时间要求
- 是否需要分页/懒加载/缓存

### 兼容性
- 浏览器/设备支持范围
- 分辨率适配策略

### 安全要求
- 数据敏感性说明
- 权限控制方式
- 加密需求

**写作要求：**
- 性能用具体数字，不用"快"、"流畅"等主观描述
- 如果某项无特殊要求，写"无特殊要求"而非省略

## AI 推断标注规范

AI 推断的内容不在正文中作为独立章节，而是采用两层标注：

### 内联注释

在相关需求描述处用 HTML 注释标注推断来源：

```markdown
列表默认按创建时间倒序排列，每页显示20条。<!-- AI-inferred: 排序规则和分页大小为推断默认值 -->
```

**标注粒度：** 只标注有业务含义的推断。纯技术决策（如 ESLint 配置、Git 分支命名）不标注。

**确认归属：** 用户在交互中对默认值的任何回应（"OK"/"可以"/"没问题"）视为用户确认，不再标注为 AI 推断。

### 附注汇总

在 spec 文件末尾（`---` 分隔线之后）附一个区域：

```markdown
---

## 附注：AI 推断项汇总

以下内容为 AI 基于项目上下文和行业最佳实践的推断，供快速 review：

| # | 功能点 | 推断内容 | 推断依据 |
|---|--------|----------|----------|
| 1 | 列表排序 | 默认按创建时间倒序 | 项目中现有列表均使用此规则 |
| 2 | 分页大小 | 每页20条 | 项目 package.json 中的全局配置 |
```

此区域是给人 review 用的，不是 writing-plans 的消费对象。writing-plans 消费 spec 时应忽略此区域。
