🚀 Quick Start
小功能、熟悉技术栈、低风险
触发条件：明确需求、小改动、无跨模块依赖
🔄 Full Flow
中等复杂度、跨模块、风险不明
触发条件：标准开发流程、需要完整质询
📋 PRD Direct
有完整 PRD/原型/明确 AC
触发条件：已有完整产品文档、需求明确
🔀 Split First
大需求含 5+ 独立功能或显式要求拆分
触发条件：复杂需求、需要拆分子任务

各阶段详细流程

0

Split - 需求拆分

ship-split (可选前置)

输入

  • 大型需求描述
  • 用户显式拆分请求
  • 5+ 独立功能点

产出

  • splits.yml (子需求清单)
  • 依赖关系图
  • TAPD/Jira 任务映射

核心职责

  • 识别独立可交付子功能
  • 建立依赖关系
  • 生成 split_id 追踪
  • 为每个子需求创建 feature 目录

完成标准

  • 每个子需求有明确边界
  • 依赖关系清晰
  • 可独立交付
1

Understand - 需求理解

ship-understand (必需)

输入

  • 口头/文字需求
  • PRD、原型、图片
  • meta.yml.scenario
  • ship-spec 加载的上下文

产出

  • requirements.md (status: ready)
  • 至少 1 个可测试 AC
  • Domain 模型
  • 非功能需求

核心职责

  • 加载 spec 上下文
  • 解析需求/PRD/原型
  • 生成 Given/When/Then AC
  • 调用 grill-me 质询 (按场景)
  • 运行 validate_requirements.py

完成标准

  • 所有 AC 使用 Given/When/Then
  • Domain 模型明确
  • 无 blocking 问题
  • validator 无 errors
2

Design - 技术设计

ship-design (必需)

输入

  • requirements.md (status: ready)
  • meta.yml.requested_design_template
  • Design 阶段 spec
  • Design Reference Template

产出

  • design.md (status: ready)
  • API Contract (Request/Response/Error)
  • 数据模型
  • 前后端方案合一
  • AC 覆盖映射

核心职责

  • 选择并引用 Design Reference Template
  • 生成合一技术设计
  • 调用 grill-me 设计审查 (按场景)
  • 记录模板偏离
  • 运行 validate_design.py
  • 等待用户确认

完成标准

  • API 契约完整 (含错误响应)
  • 每个 AC 可追踪
  • 模板引用已记录
  • 无 TBD/待补充
  • validator 无 errors
  • 用户明确确认
3

Build - 实现与测试

ship-build (必需)

输入

  • design.md (status: ready)
  • 用户确认进入 Build
  • Build 阶段 spec

产出

  • build-plan.yml (任务清单)
  • 实现代码
  • 自动化测试
  • verification.md (AC 验证报告)
  • 更新 existing-features

核心职责

  • 生成任务清单 (按依赖顺序)
  • 实现：数据层 → 后端 → 前端 → 集成
  • 运行真实测试
  • 验证 AC 覆盖
  • 运行 validate_build.py
  • 调用 ship-spec 更新知识库

完成标准

  • 所有 AC 有测试覆盖
  • 真实测试通过
  • verification.md 完整
  • validator 无 errors
  • 可交付软件

ship-grill-me 触发矩阵

场景 Understand 阶段 Design 阶段 说明
quick_start 条件触发 默认跳过 仅在发现 blocking 问题时触发 Understand；Design 跳过
full_flow 必须触发 必须触发 标准完整流程，Understand 和 Design 都必须质询
prd_direct 默认跳过 条件触发 Understand 只做提取；Design 有技术风险时触发
split_first 必须触发 必须触发 拆分后每个子需求按 full_flow 处理
