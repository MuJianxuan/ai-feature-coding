# TODO Web App — 需求清单

## 1. 需求背景与目标

构建一个简洁的 TODO Web 应用，支持用户管理日常待办事项。目标用户为个人用户，核心价值是快速记录和管理待办。

**项目目标：**
- 提供直观的 TODO 增删改查功能
- 支持 TODO 状态管理（完成/未完成）
- 支持按优先级和截止日期排序
- 响应式设计，支持移动端访问

## 2. 用户故事

| ID | 角色 | 行为 | 目的 |
|----|------|------|------|
| US-001 | 用户 | 创建一条 TODO | 记录待办事项 |
| US-002 | 用户 | 查看 TODO 列表 | 了解当前待办 |
| US-003 | 用户 | 标记 TODO 为完成 | 追踪进度 |
| US-004 | 用户 | 编辑 TODO 内容 | 修正或补充信息 |
| US-005 | 用户 | 删除 TODO | 清理不需要的条目 |
| US-006 | 用户 | 按状态筛选 TODO | 聚焦未完成事项 |
| US-007 | 用户 | 按优先级排序 | 处理重要事项 |

## 3. 功能范围

### In Scope (Must Have)

- TODO CRUD（创建、读取、更新、删除）
- TODO 状态切换（完成/未完成）
- TODO 列表展示（分页）
- 按状态筛选（全部/未完成/已完成）
- 基础表单验证

### In Scope (Should Have)

- 优先级标记（高/中/低）
- 截止日期设置
- 按优先级/日期排序

### Out of Scope

- 用户注册/登录（本期不做认证）
- 多用户协作
- 文件附件
- 提醒通知
- 数据导入导出

## 4. 业务域建模

| Domain ID | 名称 | 描述 | 核心实体 |
|-----------|------|------|---------|
| TODO | TODO 管理 | TODO 的增删改查和状态管理 | Todo |

### 实体: Todo

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string (UUID) | 是 | 唯一标识 |
| title | string | 是 | 标题，1-200 字符 |
| description | string | 否 | 详细描述 |
| completed | boolean | 是 | 完成状态，默认 false |
| priority | enum | 是 | high / medium / low，默认 medium |
| dueDate | date | 否 | 截止日期 |
| createdAt | datetime | 是 | 创建时间 |
| updatedAt | datetime | 是 | 更新时间 |

| AC-003 | TODO | 用户能标记 TODO 为完成/未完成（点击切换） | E2E 测试 |
| AC-004 | TODO | 用户能编辑 TODO 的标题、描述、优先级、截止日期 | E2E 测试 |
| AC-005 | TODO | 用户能删除 TODO（需确认弹窗） | E2E 测试 |
| AC-006 | TODO | 用户能按状态筛选（全部/未完成/已完成） | E2E 测试 |
| AC-007 | TODO | 用户能按优先级排序（高→低） | E2E 测试 |

## 6. 非功能需求

| 类别 | 要求 | 量化指标 |
|------|------|---------|
| 性能 | 列表加载时间 | < 500ms（100 条数据） |
| 性能 | 创建/更新响应时间 | < 200ms |
| 可用性 | 无障碍 | WCAG 2.1 AA 基础合规 |
| 安全 | 输入校验 | 防 XSS，服务端校验 |
| 兼容性 | 浏览器 | Chrome/Firefox/Safari 最新版 |

## 7. 约束与假设

**约束：**
- 单用户模式，无需认证
- SQLite 作为数据库（开发简单，无需额外部署）
- 前后端分离部署

**假设：**
- 数据量 < 10000 条，无需复杂分页优化
- 单机部署，无需考虑分布式
