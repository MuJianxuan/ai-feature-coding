# Discovery: 订单导出功能

## 业务背景

运营团队需要定期导出订单数据用于财务对账和数据分析。目前只能通过数据库直接查询，效率低且有安全风险。

## 用户原始描述

"我们需要一个订单导出功能，运营人员可以按时间范围、订单状态筛选后导出 Excel。数据量可能比较大，有时候一次要导几万条。"

## 逐问逐答记录

Q1: 导出的字段有哪些？
A1: 订单号、下单时间、用户名、商品名、数量、金额、订单状态、收货地址、支付方式。

Q2: 谁有权限使用这个功能？
A2: 运营主管和财务人员，大概 10 个人左右。

Q3: 导出频率？
A3: 每天都会用，月底对账时更频繁。

Q4: 有没有现成的导出框架？
A4: 没有特别调研过，你们看着选就行。

## 方案方向

后台管理系统新增导出页面，支持筛选条件 + 异步导出 + 下载。
