# Anti-Slop Checklist

> ship-shape 的 Anti-Slop 自检清单。在 design-brief.md 定稿前逐项检查。
> 每个"禁止"项如果出现在产出中，必须有明确的用户授权理由。

## 视觉模式检查

| # | 检查项 | 状态 | 备注 |
|---|--------|------|------|
| 1 | 是否使用了渐变 orb/blob 作为装饰？ | ☐ 无 / ☐ 有（已授权） | 替代：使用真实产品截图或有意义的图形 |
| 2 | 是否使用了"圆角卡片 + 左侧色条"模式？ | ☐ 无 / ☐ 有（已授权） | 替代：根据内容层次选择合适的容器样式 |
| 3 | 是否用 CSS/SVG 画了产品图或设备框？ | ☐ 无 / ☐ 有（已授权） | 替代：使用明确标注的 placeholder 色块 |
| 4 | 是否使用 emoji 作为 bullet 或 icon？ | ☐ 无 / ☐ 有（已授权） | 替代：使用 icon 库或纯文字 |
| 5 | 是否默认使用了"3 列特性网格"？ | ☐ 无 / ☐ 有（已授权） | 替代：根据内容量和层次选择布局 |
| 6 | 是否有纯装饰性渐变背景？ | ☐ 无 / ☐ 有（已授权） | 替代：纯色或有功能意义的视觉分区 |
| 7 | 是否使用了假数据/假统计数字？ | ☐ 无 / ☐ 有（已授权） | 替代：明确标注 "[placeholder]" 的示意数据 |
| 8 | 是否默认使用 Inter/Roboto/Arial？ | ☐ 无 / ☐ 有（已授权） | 替代：根据设计方向选择有辨识度的字体 |
| 9 | 是否用 SVG 剪影代替真实产品图？ | ☐ 无 / ☐ 有（已授权） | 替代：灰色色块 + 尺寸标注 |
| 10 | 是否过度 iconified（每个列表项都有图标）？ | ☐ 无 / ☐ 有（已授权） | 替代：只在有助于识别的地方使用图标 |

## 结构模式检查

| # | 检查项 | 状态 | 备注 |
|---|--------|------|------|
| 11 | 所有页面是否都长一样（同一模板复制）？ | ☐ 否 | 每个页面应根据内容需求有不同布局 |
| 12 | 是否有"hero + features + CTA"的套路首页？ | ☐ 无 / ☐ 有（合理） | 只在营销页合理；应用内页面不应如此 |
| 13 | 是否所有交互都是"点击打开 modal"？ | ☐ 否 | 考虑 inline editing、drawer、expand 等替代 |
| 14 | 导航是否只有顶部 navbar 一种形式？ | ☐ 已考虑替代 | 根据信息架构选择：sidebar、tab、breadcrumb 等 |

## 内容检查

| # | 检查项 | 状态 | 备注 |
|---|--------|------|------|
| 15 | 文案是否使用了真实语境的占位内容？ | ☐ 是 | 禁止 lorem ipsum；使用贴近真实场景的文案 |
| 16 | 空状态是否有设计（不只是"暂无数据"）？ | ☐ 是 | 空状态应引导用户下一步操作 |
| 17 | 错误状态是否有设计？ | ☐ 是 | 错误应告知原因和恢复路径 |
| 18 | 加载状态是否有设计？ | ☐ 是 | skeleton / spinner / progressive |

## 使用方式

1. 在 design-brief.md 定稿前逐项检查
2. 标记为"有（已授权）"的项必须附用户明确要求的证据
3. 将检查结果摘要写入 design-brief.md 的 "Anti-Slop Self-Check" 章节
4. 如果发现违规项且无授权，必须修改原型后再定稿
