## 审查示例 1：❌ 不应该沉淀

```markdown
规范：某个页面的表单布局

必要性检查：
- 复用次数：1 次（只有这个页面用）
- 模式稳定：是
- 跨模块：否
- 复杂度：低
→ 总分：40 分 < 60 分 ❌

结论：不应该沉淀为规范，这是一次性的实现细节
```

## 审查示例 2：✅ 值得沉淀，但需改进

```markdown
规范：API 错误处理

必要性检查：
- 复用次数：所有 API（10+ 次）
- 模式稳定：是
- 跨模块：是（前后端都用）
- 复杂度：中等
→ 总分：90 分 ✅

完整性检查：
- frontmatter ✅
- 适用场景 ✅
- 核心约束 ✅
- 示例 ⚠️（只有正确示例，缺少错误示例）
- 检查点 ✅

结论：✅ 值得沉淀，但需补充错误示例
```

## 审查示例 3：❌ 有安全问题

```markdown
规范：数据库连接配置

核心约束：
- 使用以下连接串：
  postgresql://admin:P@ssw0rd@db.prod.internal:5432/mydb

安全性检查：
- ❌ 包含密码
- ❌ 包含内部地址

结论：❌ 必须删除敏感信息，改为：
"从环境变量 DATABASE_URL 读取连接串"
```
